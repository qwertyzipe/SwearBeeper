# Swear Beeper 🔇
Пикает маты в (почти) реальном времени

Приложение, которое слушает микрофон, распознаёт мат в реальном времени и автоматически запикивает его - с небольшой задержкой (~1-2 сек), как это делают на радио и ТВ. Полезно для стримов, записи подкастов, звонков в Discord и т.д.

**Этот репозиторий содержит открытый исходный код.** Собранное приложение (`.exe`) публикуется отдельно — здесь можно посмотреть код, убедиться, что внутри нет ничего вредоносного, и при желании собрать exe самостоятельно из исходников.

## Как это работает

```
Микрофон → кольцевой буфер (полное качество, 48кГц)
         → параллельно: копия звука пересчитывается в 16кГц и уходит в Vosk (офлайн-распознавание речи)
         → если распознанное слово похоже на мат → на буфер накладывается бип
         → очищенный звук (с задержкой ~1-2 сек) уходит на выход (динамики / виртуальный микрофон)
```

Внутри задержки хватает времени, чтобы: услышать слово → распознать его → решить, мат это или нет → наложить бип **до того**, как этот кусок аудио уйдёт на выход. Со стороны это выглядит как "реальное время" с небольшим приемлемым лагом.

Распознавание речи полностью **офлайн** (через [Vosk](https://alphacephei.com/vosk/)) — никакой звук никуда не отправляется в интернет.

## Возможности

- Определение мата по словарю корней + типичных приставок (регулярные выражения, без ML-классификатора токсичности)
- Настраиваемая задержка вывода
- Настраиваемая громкость бипа
- Настраиваемый "запас" (паддинг) до и после слова, чтобы бип точно перекрывал мат целиком
- Выбор микрофона и устройства вывода прямо в интерфейсе
- Полноценное GUI-приложение (не нужно лезть в код и консоль)
- Захват и воспроизведение звука в полном качестве (48кГц) — распознавание работает с отдельной пересчитанной копией, чтобы не терять четкость голоса

## Ограничения (важно понимать)

- Это **эвристика**, а не идеальный ИИ-классификатор. Возможны как ложные срабатывания (например, слово "сукно" тоже может запикаться, так как содержит корень "сук"), так и пропуски отдельных форм слов.
- Распознавание речи не идеально — на маленькой модели Vosk (~50 МБ) возможны ошибки, особенно в шумной обстановке или с нестандартной дикцией.
- Без задержки вывода (искусственный лаг ~1-2 сек) технически невозможно "предсказать" мат до того, как он прозвучит — так работает вообще любая система живой цензуры (ровно так же, как на настоящем радио/ТВ).
- Приложение не делает шумоподавление, эхоподавление или другую доп. обработку голоса — это осознанное решение, чтобы не терять четкость распознавания.

## Требования

- Windows (тестировалось на Windows 10/11; на Linux/Mac потребуется адаптация путей к аудио-устройствам)
- Python 3.10+ (если собираешь/запускаешь из исходников)
- Модель Vosk для русского языка (см. ниже)
- Нормальный микро, ибо детект будет очень плохо работать

## Установка зависимостей (если запускаешь из исходников)

```bash
pip install sounddevice vosk numpy
```

## Модель распознавания речи (Vosk)

Модель Vosk - это папка под названием model_ru. Скачай отдельно:

- Список моделей: https://alphacephei.com/vosk/models
- Рекомендуется для старта: `vosk-model-small-ru-0.22` (~50 МБ, быстрее, чуть менее точная)
- Для лучшей точности: `vosk-model-ru-0.42` (~1.8 ГБ, точнее, но медленнее и тяжелее) (Максимально не советую так как может иметь очень большую задержку)

Распакуй модель в папку `model_ru` рядом со скриптом:

```
swear_beeper/
├── beeper_gui.py
├── model_ru/        <- сюда распакованная модель
```

## Запуск из исходников

```bash
python beeper_gui.py
```

В открывшемся окне:
1. Укажи путь к папке модели (или оставь `model_ru`, если она лежит рядом)
2. Выбери микрофон и устройство вывода из выпадающих списков
3. Настрой задержку, громкость бипа и паддинг под себя
4. Нажми "Старт"

## Использование с Discord / OBS / другими программами

Приложению нужно вывести очищенный звук в отдельное "виртуальное" устройство, которое другая программа сможет распознать как микрофон. Для этого нужен виртуальный аудио-кабель:

1. Установи бесплатный [VB-CABLE](https://vb-audio.com/Cable/) (желательно перезагрузить после установки)
2. В приложении в качестве устройства вывода выбери **CABLE Input**
3. В Discord (или другой программе): `Настройки` → `Голос и видео` → `Устройство ввода` → выбери **CABLE Output**

Цепочка: реальный микрофон → приложение слушает, распознаёт, пикает → CABLE Input → Discord слушает CABLE Output как "микрофон".

# Как скомпилировать приложение (.exe) самостоятельно

Собранное приложение можно скачать отдельно (см. Releases), но если хочешь убедиться, что exe действительно собран из этого кода без ничего лишнего — вот как собрать самому.

## 1. Установи PyInstaller

```bash
pip install pyinstaller
```

## 2. Подготовь модель

Положи распакованную модель Vosk в папку `model_ru` рядом с `beeper_gui.py` (см. раздел про модель выше). Для сборки exe рекомендуется маленькая модель (~50 МБ).

## 3. (Опционально) подготовь иконку

Иконка должна быть в формате `.ico`. Если у тебя `.png`:

```bash
pip install Pillow
python -c "from PIL import Image; Image.open('icon.png').save('icon.ico')"
```

Положи `icon.ico` рядом со скриптом.

## 4. Собери exe

**Базовая команда** (модель встраивается прямо в exe, один файл):

```bash
pyinstaller --onefile --windowed --name SwearBeeper --add-data "model_ru;model_ru" --collect-all vosk beeper_gui.py
```

**С иконкой:**

```bash
pyinstaller --onefile --windowed --name SwearBeeper --icon=icon.ico --add-data "model_ru;model_ru" --collect-all vosk beeper_gui.py
```

Флаги:
- `--onefile` — всё упаковывается в один exe-файл
- `--windowed` — без консольного окна (только GUI)
- `--name SwearBeeper` — имя итогового файла
- `--add-data "model_ru;model_ru"` — встраивает модель Vosk внутрь exe (разделитель `;` — только для Windows; на Linux/Mac был бы `:`)
- `--collect-all vosk` — обязательно, иначе PyInstaller не подхватит нативные библиотеки Vosk и приложение упадёт с ошибкой при запуске
- `--icon=icon.ico` — иконка приложения (опционально)

## 5. Готово

Итоговый файл — `dist/SwearBeeper.exe`. Он полностью самодостаточен: содержит и код, и модель распознавания речи, ничего рядом класть не нужно.

## Если сборка ругается на ошибки

**Ошибка про `vosk` / нативные библиотеки при запуске exe:**
```bash
pyinstaller --onefile --windowed --name SwearBeeper --add-data "model_ru;model_ru" --collect-all vosk --clean beeper_gui.py
```
`--clean` очищает кэш предыдущей сборки — полезно, если пересобираешь несколько раз подряд с разными флагами.

**Ошибка про `sounddevice` / PortAudio:**
```bash
pyinstaller --onefile --windowed --name SwearBeeper --add-data "model_ru;model_ru" --collect-all vosk --collect-all sounddevice beeper_gui.py
```

**Антивирус ругается на собранный exe:**
Это частая ложная тревога для любых `.exe`, собранных PyInstaller с флагом `--onefile` — код самораспаковывается во временную папку при запуске, что похоже на поведение некоторых вредоносов по паттерну поведения, а не по содержимому. Это не связано с содержимым кода — можешь убедиться в этом, прочитав исходники в этом репозитории.
